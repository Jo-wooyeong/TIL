## 라우팅
```
보통 프로젝트 루트의 routes 디렉토리에 있는 파일들을 사용하며, 가장 핵심은 routes/web.php입니다.
```

### web.php 작성
```php
use App\Http\Controllers\GlobalSupportController;

// 'example.com/welcome' 접속 시 welcome.blade.php 뷰를 보여줌
Route::get('/welcome', function () {
    return view('welcome');
});
```

#### ->controller
`->controller`를 사용해서 함수를 불러올 컨트롤러를 한꺼번에 사용할 수 있습니다. <br />
예시는 `->group`과 함께 볼 수 있습니다.
```php
// controller 사용 안할때
Route::get('/posts', [PostController::class, 'index']);
Route::get('/posts/{id}', [PostController::class, 'show']);
Route::post('/posts', [PostController::class, 'store']);
```

#### ->group 을 이용
```php
use App\Http\Controllers\GlobalSupportController;

Route::middleware(['auth'])
    ->controller(GlobalSupportController::class)
    ->group(function () {
        $router->get('/', 'home')->name('home');

        $router->get('/service/{serviceId}', 'service')->name('service-detail');

        $router->get('/evangelist', 'evangelist')->name('evangelist');
        $router->get('/evangelist/new', 'createEvangelist')->name('create-evangelist');
        $router->get('/evangelist/{evangelistId}', 'evangelistDetail')->name('evangelist-detail');
        // 이 안의 모든 라우트는 로그인(auth)이 되어 있어야 접속 가능합니다.
});
```

### 컨트롤러 작성
라우트 시 필요한 함수를 작성합니다. return전에 앤티티를 불러와 SSR방식으로 서버에서 데이터를 먼저 받아와서 `props`로 넘겨줄 수 있습니다.
```php
public function evangelistDetail($evangelistId): Response
  {
      $post = EvangelistBoard::findOrFail($evangelistId);

      if (isset($post->user_id)) {
          $post->setRelation('user', \App\Models\User::find($post->user_id));
      }

      return InertiaRender::vue('Evangelist/EvangelistDetail', [
          'evangelistId' => $evangelistId
      ]);
  }
```
