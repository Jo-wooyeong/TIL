## INNER_JOIN
```
두 테이블 간 공통된 값을 가진 행들만 합쳐서 보여주는 방식(교집합)
JOIN과 완전히 동일합니다
```

```sql
SELECT 
    A.컬럼명, 
    B.컬럼명
FROM 테이블_A A
INNER JOIN 테이블_B B
    ON A.기준키 = B.기준키; -- 두 테이블을 연결하는 '연결고리'
```

### 최적화 tip
- 인덱스(Index) 활용: ON 절에 사용되는 컬럼(기준키)에는 반드시 인덱스가 걸려 있어야 합니다. 인덱스가 없다면 DB는 두 테이블을 한 줄씩 다 대조하는 최악의 작업(Full Scan)을 수행하게 됩니다.
- 드라이빙 테이블(Driving Table): 데이터 양이 적은 테이블을 먼저 읽고, 큰 테이블을 나중에 읽는 것이 효율적입니다. (최신 SQL 옵티마이저는 자동으로 처리해주기도 하지만, 쿼리를 짤 때 이를 염두에 두는 습관이 중요합니다.)
- ON vs WHERE: 조인 조건은 반드시 ON 절에 적으세요. 필터링 조건은 WHERE에 적는 것이 가독성과 실행 계획 수립에 유리합니다.
