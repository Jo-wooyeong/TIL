## INNER_JOIN
```
두 테이블 간 공통된 값을 가진 행들만 합쳐서 보여주는 방식(교집합)
JOIN과 완전히 동일합니다
```

```sql
SELECT 
    A.컬럼명, 
    B.컬럼명
FROM 테이블_A A
INNER JOIN 테이블_B B
    ON A.기준키 = B.기준키; -- 두 테이블을 연결하는 '연결고리'
```

### 최적화 tip
- 인덱스(Index) 활용: ON 절에 사용되는 컬럼(기준키)에는 반드시 인덱스가 걸려 있어야 합니다. 인덱스가 없다면 DB는 두 테이블을 한 줄씩 다 대조하는 최악의 작업(Full Scan)을 수행하게 됩니다.
- 드라이빙 테이블(Driving Table): 데이터 양이 적은 테이블을 먼저 읽고, 큰 테이블을 나중에 읽는 것이 효율적입니다. (최신 SQL 옵티마이저는 자동으로 처리해주기도 하지만, 쿼리를 짤 때 이를 염두에 두는 습관이 중요합니다.)
- ON vs WHERE: 조인 조건은 반드시 ON 절에 적으세요. 필터링 조건은 WHERE에 적는 것이 가독성과 실행 계획 수립에 유리합니다.

## LIMIT
```
쿼리 결과로 나온 전체 레코드 중에서 특정 개수만큼만 결과로 반환하라는 명령어입니다. 쿼리의 가장 마지막에 위치해야 합니다.
```

| 구문 | 역할 | 예시 |
|---|---|---|
| LIMIT n | 상위 n개 조회 | LIMIT 5 (1~5위) |
| LIMIT n OFFSET m | m개 건너뛰고 n개 조회 | LIMIT 5 OFFSET 10 (11~15위) |
| LIMIT m n | m번째부터 n개 (MySQL 전용) | LIMIT 10, 5 (11~15위) |

테이블에 데이터가 얼마나 있는지 모르는 상태에서 SELECT *를 날리는 건 위험합니다. 항상 LIMIT 10 정도를 붙여서 데이터의 모양새를 먼저 확인하는 습관을 들이세요.

## BETWEEN
```
WHERE 조건을 쓸 때 사이 값을 쓰고 싶을 떄
```

`TABLE`테이블에서 `AGE`값이 20에서 29사이인 값
- `WHERE AGE >= 20 AND AGE <= 29`와 같은 의미
```sql
SELECT COUNT(*) AS USERS FROM TABLE
WHERE AGE BETWEEN 20 AND 29
```
