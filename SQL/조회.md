## SELECT, FROM
```
출력하고자하는 컬럼을 선택한다. AS 키워드를 이용해서 원하는 컬럼명으로 변경 가능하다.
검색할 테이블명을 FROM 키워드 뒤에 작성한다.
```

```sql
SELECT
	USER_ID,
    TITLE AS USER_NAME,
    DATETIME,
FROM USER;
```

## WHERE
```
테이블 전체 데이터 중 조건에 맞는 행만 골라낸다.
예) WHERE CATEGORY='인문' : 카테고리가 인문인 책만 남기고 나머지는 다 버림
```

```sql
SELECT * FROM BOOK
WHERE CATEGORY='인문';
```

#### AND, OR, LIKE
```
주로 WHERE과 함께 사용하며 LIKE는 '%', '_' 두개를 사용해 조건을 건다
```

```sql
-- 010으로 시작하는 데이터
SELECT * FROM USER
WHERE PHONE LIKE '010%'

-- 네자리 중 끝자리가 a인 데이터 (_ 하나에 한 자리)
WHERE FIRST_NAME LIKE '___a'
```

## GROUP BY(그룹화)
```
특정 컬럼의 값이 같은 데이터들을 하나의 덩어리로 묶는다.
보통 COUNT(), SUM(), AVG() 같은 집계 함수와 함께 사용된다.
예) GROUP BY CATEGORY='인문': 카테고리가 인문인 그룹과 인문이 아닌 그룹 두개로 데이터를 쪼갬
```

```sql
SELECT * FROM BOOK
GROUP BY CATEGORY='인문';
```

## HAVING
```
그룹에 조건걸기
```

그룹 함수는 `COUNT`, `MAX`, `MIN`, `AVG`, `SUM`, `VARIANCE(분산)`, `STDDEV(표준편차)`가 있다.
```sql
SELECT * FROM BOOK
GROUP BY CATEGORY='인문'
HAVING COUNT(*) >= 2; -- 갯수가 2이상인 그룹
```

## ORDER BY(정렬)
```
최종 결과 데이터를 정렬하는 법.
기본: 오름차순, DESC: 내림차순(역순)
예) ORDER BY BOOK_ID, DATETIME DECS: 아이디순으로 순서대로 정렬(오름차순), 아이디가 같으면 구매일 역순(내림차순)
```

```sql
SELECT * FROM BOOK
WHERE CATEGORY='인문'
ORDER BY BOOK_ID, DATETIME DECS;
```

### IS NOT NULL
```Md
조건문에서 NULL이 아닌것을 조회 라는 조건을 달때 `IS NOT NULL`을 사용한다.
```

`NULL`은 값이 아니라 상태이기 떄문에 `AGE != NULL`과 같이 비교할 수 없다.
