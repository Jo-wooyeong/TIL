## Entity
```
외부 API로부터 데이터를 받아와 데이터와 관련된 처리, 계산로직 등을 설계하는 모델
아래는 앤티티를 구성하는 예시를 나타냈습니다.
```

### 상속
기본적으로 `entities/` 내부 앤티티는 모두 `BaseEntity.ts`를 상속받아 사용한다.
모든 앤티티에서 사용하는 공통 로직 작성 대비
```ts
export default class BaseEntity{}
```

```ts
export default class PlanEntity extends BaseEntity {}
```

### 프로퍼티 정의
클래스 최상단에 변수 이름을 적는다. 이 객체가 어떤 데이터를 가질 것인지 미리 선언
```ts
export default class PlanEntity extends BaseEntity {
    // 이 객체는 아래 4가지 데이터를 반드시 가집니다.
    country: string;
    region: string;
    company: string;
    imageUrl: string;
}
```

### 생성자
`new`키워드를 통해 객체가 생성될 때 가정 먼저 실행되는 함수. 원재료(data)를 프로퍼티라는 방에 배정 (data 타입은 `/types`에서 정의)<br />
`constructor(){}`를 통해 생성자를 생성한다. 부모의 생성자를 이용할 경우 반드시 `super()`를 작성해야한다.
```ts
export default class PlanEntity extends BaseEntity {
    // ...
    constructor(data: Plan){
        super();

        // 외부에서 받은 data를 클래스 내부(this) 프로퍼티에 저장
        this.country = data.country;
        this.region = data.region;
        this.company = data.company;
        this.imageUrl = data.imageUrl;
    }
}
```

### 비즈니스 로직
예를들어 앤티티의 `fromJson()` 함수는 api를 통해 받아온 스네이크케이스의 변수명을 클라이언트에서 사용하는 카멜케이스로 바꿉니다.<br />
호출 후 날것의 데이터를 이용하는 기본 로직 등을 작성
```ts
export default class PlanEntity extends BaseEntity {
    // ...
    static fromJson(json: Record<string, any>): PlanEntity{
        return new PlanEntity({
            announceMonthlyPrice: json.announce_monthly_price,
            monthlyPrice: json.monthly_price,
            title: json.title,
            evoTrack: {
                equipment: json.evo_track?.equipment || 0,
                announceEquipment: json.evo_track?.announce_equipment || 0,
            },
                solutionEssentials: {
                paseModel: json.solution_essentials?.pase_model ?? null,
                setupCostPerCorporation: json.solution_essentials?.setup_cost_per_corporation ?? "",
                setupCostPerLine: json.solution_essentials?.setup_cost_per_line ?? "",
                infrastructureCost: json.solution_essentials?.infrastructure_cost ?? "",
                dataStorageDuration: json.solution_essentials?.data_storage_duration ?? null,
                dwDataStorageDuration: json.solution_essentials?.dw_data_storage_duration ?? null,
            },
            
            serverRequirements: {
                requiredServer: json.server_requirements?.required_server ?? null,
                osSpec: json.server_requirements?.os_spec ?? null,
                cpu: json.server_requirements?.cpu ?? null,
                memory: json.server_requirements?.memory ?? null,
                dwStorage: json.server_requirements?.dw_storage ?? null,
                apStorage: json.server_requirements?.ap_storage ?? null,
            },
        })
    }
}
```
